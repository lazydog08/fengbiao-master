# 封标大师产品简报

## 背景

小黑的内容主阵地是 B 站和抖音，其中 B 站视频点击很大程度由封面和标题共同决定。当前需求不是简单下载素材，也不是用绝对播放量反推“为什么能火”，而是长期积累高质量创作者样本，形成一个能解释、能检索、能辅助决策的封标知识库。

## 核心用户

- 内容创作者：需要判断某个选题的封面和标题如何表达。
- 研究助手：需要定期整理优秀博主的新视频表现。
- 未来的建议引擎：根据数据库样本给出可执行的封标建议。

## 第一阶段能力

1. 管理 UP 主清单。
2. 定时抓取公开视频公开元数据。
3. 记录播放量、粉丝数和封面/标题随时间变化。
4. 计算相对 UP 主自身基线倍数、播放/粉丝比和前 48 小时增速。
5. 缓存封面图片，保留原始链接、图片哈希和尺寸。
6. 提供样本卡参考墙：标题、封面、相对表现、赛道标签和一句话人工判断。

## 第二阶段能力

1. 标题结构拆解：利益点、冲突点、悬念、对象、人群、场景。
2. 同赛道对比：跑赢基线样本和低于基线样本差在哪里。
3. 封面/标题发布后变更追踪：识别改封面、改标题之后播放曲线是否变快。
4. 建议生成：输入选题后，输出标题方向、封面视觉方向、风险点和参考样本。
5. 封面图片自动理解：OCR、主体识别、色彩和构图特征，放到样本稳定之后再做。

## 采集原则

- 只采集公开视频公开可见数据。
- 不保存账号 cookie、浏览器资料、私信、评论隐私或任何登录态凭证。
- 不绕过验证码、不突破频率限制。
- 请求速率要保守，失败要退避。
- 每条数据保留来源 URL、采集时间和采集方法。

## 初始数据模型草案

- `creators`：UP 主基础信息、分类标签、是否启用。
- `videos`：视频 ID、标题、封面 URL、发布时间、分区、UP 主、初始标题哈希、初始封面哈希。
- `video_snapshots`：采集时间、播放量、粉丝数、点赞、投币、收藏、分享、评论、弹幕。
- `video_metrics`：相对自身基线倍数、播放/粉丝比、前 48 小时增速、是否跑赢基线。
- `cover_assets`：本地封面路径、哈希、尺寸、主色、OCR 文本。
- `annotations`：人工或自动标签，例如选题类型、标题结构、钩子类型、封面主体、价格锚点、是否有反差。
- `change_events`：标题或封面发生变化的时间、变化前后哈希、变化后的播放曲线。
- `analysis_runs`：每次分析任务的输入、模型版本、输出摘要。

## 成功标准

第一期成功不是“能生成很玄的建议”，而是能稳定留下可追溯的数据，并能按一个选题赛道调出至少 5 张跑赢自身基线的样本卡：每张卡包含封面、标题、相对表现、早期增速和一句话人工判断。这个指标比“攒了多少条播放量快照”更接近创作者真实决策。
