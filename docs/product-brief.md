# 封标大师产品简报

## 背景

小黑的内容主阵地是 B 站和抖音，其中 B 站视频点击很大程度由封面和标题共同决定。当前需求不是简单下载素材，而是长期积累高质量创作者样本，形成一个能解释、能检索、能辅助决策的封标知识库。

## 核心用户

- 内容创作者：需要判断某个选题的封面和标题如何表达。
- 研究助手：需要定期整理优秀博主的新视频表现。
- 未来的建议引擎：根据数据库样本给出可执行的封标建议。

## 第一阶段能力

1. 管理 UP 主清单。
2. 定时抓取公开视频公开元数据。
3. 记录播放量和互动指标随时间变化。
4. 缓存封面图片，保留原始链接、图片哈希和尺寸。
5. 提供基础分析：标题长度、关键词、发布时间、封面文字、人物/产品/场景等标签。

## 第二阶段能力

1. 封面图片自动理解：OCR、主体识别、色彩和构图特征。
2. 标题结构拆解：利益点、冲突点、悬念、对象、人群、场景。
3. 爆款样本聚类：同类选题里哪些表达更容易获得高点击。
4. 建议生成：输入选题后，输出标题方向、封面视觉方向、风险点和参考样本。

## 采集原则

- 只采集公开视频公开可见数据。
- 不保存账号 cookie、浏览器资料、私信、评论隐私或任何登录态凭证。
- 不绕过验证码、不突破频率限制。
- 请求速率要保守，失败要退避。
- 每条数据保留来源 URL、采集时间和采集方法。

## 初始数据模型草案

- `creators`：UP 主基础信息、分类标签、是否启用。
- `videos`：视频 ID、标题、封面 URL、发布时间、分区、UP 主。
- `video_snapshots`：采集时间、播放量、点赞、投币、收藏、分享、评论、弹幕。
- `cover_assets`：本地封面路径、哈希、尺寸、主色、OCR 文本。
- `annotations`：人工或自动标签，例如选题类型、标题结构、封面元素。
- `analysis_runs`：每次分析任务的输入、模型版本、输出摘要。

## 成功标准

第一期成功不是“能生成很玄的建议”，而是能稳定留下可追溯的数据：给一批博主，系统能每天/每几小时记录新视频和播放量快照，并能按选题、UP 主、时间、播放增长筛出值得复盘的案例。
